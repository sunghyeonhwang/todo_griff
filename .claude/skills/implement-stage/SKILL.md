---

인자로 받은 스테이지 번호 n에 대해:

1. `DESIGN.md` §10에서 스테이지 n의 목표·핵심 산출·수용 기준을 재확인하고, `PROGRESS.md`에서 선행 스테이지(1..n-1)가 완료됐는지 확인한다. 미완료 선행이 있으면 사용자에게 알리고 중단.
2. Agent 도구로 **implementer**(스테이지 4·5는 **gesture-engineer**)를 디스패치한다. 프롬프트에 스테이지 번호·목표·파일 목록·수용 기준을 포함.
3. 구현 완료 후 **ui-verifier**를 디스패치해 해당 스테이지 수용 체크리스트를 실제 브라우저에서 검증한다.
4. FAIL 항목이 있으면 실패 표를 구현 에이전트에 피드백해 수리시킨다(최대 2회 반복). 그래도 실패하면 실패 표를 사용자에게 보고하고 중단.
5. 전부 PASS면: PROGRESS.md 갱신 상태 확인, 스테이지 커밋 존재 확인 후 체크리스트 결과 표를 출력하고, 사용자에게 실행 중인 앱을 직접 확인해 보라고 안내한 뒤 다음 스테이지 진행 여부를 묻는다.
